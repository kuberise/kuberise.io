## Steps

### 1. Fork and Clone

Fork [kuberise/kuberise.io](https://github.com/kuberise/kuberise.io) to your GitHub account, then clone it:

```bash
git clone https://github.com/<your-username>/kuberise.io.git
cd kuberise.io
```

### 2. Run the Install Script

```bash
./scripts/install.sh
  --context <CONTEXT> \
  --cluster <NAME> \
  --repo <REPO_URL> \
  --revision <REVISION> \
  --domain <DOMAIN> \
  [--token <TOKEN>]
```

| Flag | Description |
|------|-------------|
| `--context` | **(required)** Kubernetes context name (`kubectl config current-context`) |
| `--repo` | **(required)** URL of your forked repository |
| `--cluster` | Cluster name - must match a `values-<NAME>.yaml` file and a `values/<NAME>/` folder (default: `onprem`) |
| `--revision` | Branch, tag, or commit SHA, e.g. `main` (default: `HEAD`) |
| `--domain` | Base domain for services, e.g. `onprem.kuberise.dev` (default: `onprem.kuberise.dev`) |
| `--token` | Git token for private repos (optional) |

Run `./scripts/install.sh --help` for the full list of flags and environment variables.

### 3. Example

```bash
./scripts/install.sh \
  --context minikube \
  --cluster onprem \
  --repo https://github.com/<your-username>/kuberise.io.git \
  --revision main \
  --domain onprem.kuberise.dev
```

The script will:
1. Validate required CLI tools and cluster connectivity
2. Create namespaces and secrets
3. Generate CA certificates
4. Install Cilium (CNI)
5. Install ArgoCD
6. Create the root app-of-apps ArgoCD Application
7. Configure OAuth2 client secrets

## Post-Installation

After installation, access dashboards via subdomains of your domain:

- ArgoCD: `https://argocd.<DOMAIN>`
- Grafana: `https://grafana.<DOMAIN>`

```bash
./scripts/uninstall.sh <CONTEXT> <NAME>
```
